# Slice 2 — The Brief (implementation contract)

**Status:** Ready to build · **Depends on:** Slice 0 (design system) ✓ · **Independent of:** Slice 1 (Timeline) — no shared surface beyond stable primitives, so it can proceed in parallel.
**Source of truth:** `docs/ui-ux-revamp-plan.md` §7A, §7 A′, and the §16 hardening criteria. This spec operationalizes them.

---

## 1. Goal

Reconceive the dashboard into **The Brief**: the five-second answer to *"are we closing on time, and what's in my way?"* It opens with a **synthesized serif headline**, pairs it with the readiness ring and the single next best action, surfaces three instrument reads, and drills into **The Closing Pack** for the full go/no-go detail. Setup work (editing deal meta / Closing Date X) leaves the stage.

The intelligence already exists in `lib/rules.ts` (`getReadiness`) and `lib/dateUtils.ts`. This slice is about **presenting** it as a briefing — not new computation, and **not** new free-text input.

---

## 2. Scope

**In:** the Brief surface, the synthesized headline engine, the three instrument reads, the Closing Pack drill-in (`ClosingReadiness.tsx` → `ClosingPack.tsx`), and relocating deal-setup to a Deal Settings affordance.

**Out (do not build here):** the full `⌘K` command palette (Slice 3 — leave a single registration hook, see §7), any checklist/run-sheet work (Slice 3), data-model or Supabase changes, any new narrative/free-text field.

---

## 3. The synthesized headline (the centerpiece)

A deterministic, **template-driven** sentence built **only** from structured signals — never from `task.notes`, `task.comments`, or any free text. Wording/voice may be tuned, but the inputs are fixed and the output is generated by code, not typed by a user.

### Inputs (the allowlisted projection)
From a **single** `getReadiness(deal)` plus `lib/dateUtils`:
`deal.name` · `readiness.ready` · `readiness.score` · `blockers` count · `overdue` count · `criticalOpen` count · `statutoryOnClock` count (filing present & not complete) · `daysToClose` (`daysBetween(closingDateX, today)`, may be negative) · `closingDateX`.

### Disposition (decision table)
| Condition | Disposition |
|---|---|
| no `closingDateX` | `unset` |
| `readiness.ready` | `ready` (wins even if the date has passed) |
| `daysToClose < 0` && not ready | `overdue` |
| not ready && (`blockers > 0` or `overdue > 0`) | `at-risk` |
| otherwise | `on-track` |

### Sentence assembly (two parts: disposition + obstacle clause)
- **unset:** `"Set Closing Date X to generate {name}'s closing brief."` (no obstacle clause; render the onboarding state, §6)
- **ready:** `"{name} is ready to close{ on {Xdate} | — X was {Xdate}}. Nothing stands between you and a clean close."`
- **on-track:** `"{name} is tracking to close on {Xdate}."` + obstacle clause
- **at-risk:** `"{name}'s {Xdate} close is at risk."` + obstacle clause
- **overdue:** `"{name}'s {Xdate} close date has passed with items still open."` + obstacle clause

**Obstacle clause** — compose from non-zero counts, natural-join with commas + "and", suffix "stand between you and a clean close.":
`{blockers} blocker(s)` · `{statutoryOnClock} statutory filing(s)` · `{overdue} overdue item(s)` (omit the overdue part when disposition is already `overdue`). If every part is zero → `"Nothing stands in the way."`

### Worked examples (demo data: X = 30 May 2026, today after X, not ready)
- **overdue:** *"Aarohan Labs' 30 May 2026 close date has passed with items still open — 13 blockers and 2 statutory filings stand between you and a clean close."*
- **at-risk:** *"Aarohan Labs' 18 Jun 2026 close is at risk — 13 blockers and 16 overdue items stand between you and a clean close."*
- **ready:** *"Aarohan Labs is ready to close on 30 May 2026. Nothing stands between you and a clean close."*

Render via `BriefHeadline` (serif, `font-display`). Keep it to one or two sentences.

## 4. Composition (top to bottom)

1. **Masthead** — deal name + one-line context (reuse existing header; this lives in `ClosingRoomApp`).
2. **Brief hero** — `ReadinessRing` (keep) paired with the `BriefHeadline`. The ring is the click target into the Closing Pack (§5).
3. **Next best action** — the existing directive line, already using `TaskRef` + the priority/owner/deadline badges. Keep; it's the DNA of the whole product.
4. **Three instrument reads** — a row of three compact cards, each = a count + the single most-urgent example (`TaskRef`) + a drill-in:
   - **Blockers** — `readiness.blockers.length`; example = highest-priority open blocker → drills to a blocker-filtered view.
   - **Statutory clock** — next statutory hard-stop (form + date + countdown via `CountdownTile`/`DeadlinePair` language) → drills to the statutory view/Timeline.
   - **Critical path** — `criticalOpen` count; example = nearest critical item → drills to the critical path.
5. **(removed)** the "Deal details & Closing Date X" form — relocated to Deal Settings (§5).

Keep it calm: synthesized prose + a few numbers, generous whitespace, no grids on this surface.

## 5. The Closing Pack drill-in + Deal Settings relocation

**Closing Pack** — `components/checklist/ClosingReadiness.tsx` → `ClosingPack.tsx`:
- Restyle into the Editorial Chambers language as a **go/no-go board** with a `Masthead`.
- **Preserve every bucket**: Pending CPs, Mandatory incomplete, Missing evidence, Missing agreed form, Waived CPs, Converted to CS — **plus** the Final Closing Pack Checklist. Nothing drops.
- Reads from the **same** `getReadiness(deal)` result — no independent recompute.
- Reachable from: the Brief's readiness ring (click), and a hook for the future `⌘K` "Open closing pack" (§7).

**Nav wiring** (`components/ClosingRoomApp.tsx`):
- Rename the `Dashboard` view/label to **Brief**; rename `Readiness` to **Closing Pack**.
- Clicking the readiness ring on the Brief sets the active view to **Closing Pack**.

**Deal Settings** — relocate the deal-setup form (name / company / investor / Closing Date X) off the Brief:
- Preferred: a lightweight **Deal Settings** view/panel reached from a small gear/"Settings" control in the header.
- Acceptable fallback if nav plumbing is heavy this slice: a clearly **secondary, de-emphasized** "Deal settings" card at the very bottom of the Brief, visually separated from the brief content — but the headline/ring/reads must own the top of the surface.
- Closing Date X is still settable from the no-X onboarding state (§6) and editable in Deal Settings.

---

## 6. States

- **No Closing Date X (first-run / onboarding):** the Brief renders a clean fallback whose **single CTA sets Closing Date X** (mirror `TimelineOnboarding`). Not an error state — it's onboarding. The headline uses the `unset` template.
- **Ready:** `ready` disposition; ring green; reads collapse to "clear".
- **Normal / at-risk / overdue:** as per §3.
- Every state reads from one `getReadiness(deal)`.

## 8. Confidentiality contract (binding)

Synthesis is an **allowlisted typed projection** of structured fields only (names, counts, statuses, enums, computed dates, owners). The synthesis path in `lib/brief.ts` must be **structurally unable** to read `task.notes` / `task.comments` / any free text. A `grep` of `lib/brief.ts` for `.notes`/`.comments` returns nothing. This is the spine — the briefing aesthetic must never become a free-text surface.

---

## 9. Acceptance criteria

- [ ] The Brief opens with the synthesized serif headline; it changes correctly across `ready` / `on-track` / `at-risk` / `overdue` / `unset` (verify by toggling Closing Date X and a few statuses in the demo).
- [ ] Headline + reads + ring + Closing Pack all read from **one** `getReadiness(deal)` call — no surface recomputes readiness independently.
- [ ] Three instrument reads render (count + most-urgent `TaskRef` example + drill-in).
- [ ] Readiness ring click opens the **Closing Pack**, which preserves **all six buckets + the Final Closing Pack Checklist**.
- [ ] Deal-setup form no longer occupies the Brief stage; it lives in Deal Settings (or the de-emphasized fallback).
- [ ] No-X state shows the onboarding fallback with the single "Set Closing Date X" CTA.
- [ ] **Confidentiality:** `lib/brief.ts` cannot and does not touch free text (grep clean).
- [ ] `npm run typecheck`, `npm run lint`, `npm run build` green.
- [ ] Visual pass (light + dark): headline legible in serif; ring + reads cohesive; Closing Pack reads as a go/no-go board.
